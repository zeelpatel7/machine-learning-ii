Part B: Conceptual Questions¶

1. Why is it important to have a test set?¶

Having a test set is crucial because it gives us a realistic measure of how our model will perform on unseen data. When we train a model, it learns patterns from the training data — but if we only evaluate it on that same data, we can’t really tell if the model has actually learned something useful or if it’s just memorizing. The test set acts like a simulation of future data, helping us check whether the model will generalize well. Without a test set, we risk being overly optimistic about model performance and might deploy something that doesn’t actually work in practice.

2. Why is regularization important? What happens if you do not do it?¶

Regularization is important because it helps prevent overfitting, especially when we have many features or correlated predictors. It does this by adding a penalty to large coefficient values, which encourages the model to keep things simpler and more robust.

If we don’t use regularization, especially in models like linear regression, the model can end up fitting noise instead of signal — meaning it performs well on training data but poorly on new, unseen data. This is particularly risky when there’s multicollinearity or when the dataset is small compared to the number of features.

3. As we move further into Machine Learning, we will need to continually consider the bias-variance tradeoff. Explain what bias is and what variance is in regards to the bias-variance tradeoff.¶

Bias is the error from wrong assumptions in the model. A high-bias model is too simple to capture the underlying trend in the data — it underfits. For example, fitting a straight line to a clearly curved pattern.

Variance is the error from the model being too sensitive to fluctuations in the training data. A high-variance model learns too much detail (including noise), so it performs well on training data but poorly on new data — this is overfitting.

The bias-variance tradeoff is all about finding a sweet spot — a model that’s flexible enough to capture real patterns (low bias), but not so flexible that it memorizes noise (low variance).

4. In a linear regression model, how might we reduce bias?¶

To reduce bias in a linear regression model, we can:

  • Add more relevant features that help explain the outcome better

  • Consider polynomial or interaction terms if we suspect non-linearity

  • Use more complex models if the true relationship isn’t linear

Basically, we reduce bias by allowing the model to become more expressive, as long as we’re not overcomplicating it to the point of overfitting.

5. In a linear regression model, how might we reduce variance?¶

To reduce variance, we can:

  • Use regularization techniques like Ridge or Lasso

  • Reduce the number of features (especially irrelevant or noisy ones)

  • Increase the size of the training dataset (if possible)

  • Use simpler models or restrict model flexibility

These steps help the model focus on core patterns instead of noise, making it more stable across different datasets.
